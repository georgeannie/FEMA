• Request interval – arrival time between two consecutive requests.
  • Time to fulfill request – this represents the time it takes from a request being submitted in the system to issuing a responder deployment for that request, at which point the request is considered fulfilled. It is calculated as deployment-start-time minus request-received-time.
  • Request execution time – time it takes for a request to be completed. It is calculated as deployment-end-time minus deployment-start-time.
  • Bench time – waiting time for a responder between the end of one task and the start of the next task assigned to the same responder. It is calculated as deployment-start-time minus deployment-end-time of previous task done by the same responder.
  • Responder utilization rate is annualized and is defined as the number of days a responder is deployed divided by the number of days in a calendar year. Fractional days may be included in this calculation.
  • Bench rate is annualized and is defined as the number of days a responder is NOT deployed divided by the number of days in a calendar year (Total Bench time divided by the number of days).
  • Cost Ratio is the ratio of the cost of an hour of deployment fulfillment delay (time to fullfill request) to the costs of an hour of a responder bench time.
